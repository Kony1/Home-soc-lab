## 🛡️ MITRE ATT&CK – Defense Evasion: Disable Security Tools (T1562.001)

### 🎯 Tactic: Defense Evasion (TA0005)

### 🔧 Technique: T1562.001 – Impair Defenses: Disable Security Tools

### 🖥️ Platform: Linux (Ubuntu)

### 📡 Detection Sources: Syslog, Wazuh, Microsoft Sentinel (AMA)

## 📝 1. Popis techniky

Útočník, který již získal přístup do systému (např. přes SSH klíč nebo heslo), se často pokouší vypnout bezpečnostní služby, aby:

skryl svoji aktivitu

zabránil logování

ztížil detekci

připravil si prostor pro další útoky

Na Linuxu jsou nejčastější cíle:

auditd – auditní framework

rsyslog – hlavní logovací služba

Pokus o jejich vypnutí je jasný indikátor Defense Evasion.


## 🧨 2. Útok (simulace)

🔥 Pokus o vypnutí auditd

```
sudo systemctl stop auditd
```

🔥 Pokus o vypnutí rsyslog

```
sudo systemctl stop rsyslog
```

Oba příkazy spadají pod jednu MITRE techniku – T1562.001.




## 🔍 3. Detekce

### 🛡️ Wazuh

Wazuh zachytil spuštění příkazu systemctl stop auditd přes sudo. Jedná se o pokus útočníka vypnout bezpečnostní službu auditd, což je indikátor techniky T1562.001 – Disable Security Tools.

### ☁️ Microsoft Sentinel (Syslog)

### 🔎 Detekce pokusu o vypnutí auditd / rsyslog

kontrola v Sentinel logech neodhalila události zastavení logovacích služeb, pravděpodobně kvůli tomu, že služba běžela krátce a logy nebyly doručeny

```
Syslog
| where SyslogMessage contains "stop"
| where SyslogMessage contains "auditd" or SyslogMessage contains "rsyslog"
| project TimeGenerated, HostName, ProcessName, SyslogMessage
```

## 🧠 5. Analyst Notes (SOC pohled)

Pokus o vypnutí auditd nebo rsyslog je silný indikátor útoku

Legit procesy tyto služby téměř nikdy nezastavují

Útočník se snaží skrýt aktivitu nebo připravit další fázi útoku

Doporučená reakce:

ověřit předchozí SSH přihlášení

zkontrolovat změny v authorized_keys

prověřit další příkazy v Syslogu

izolovat stroj, pokud je podezření potvrzeno


## 🛡️ 6. Hardening / Remediace


Monitorovat systemd stop/start události

Omezit sudo práva

Povolit MFA / FIDO2 pro SSH

## Závěr
Simulace úspěšně demonstruje techniku T1562.001 – Disable Security Tools.
Wazuh zachytil pokus o vypnutí bezpečnostních služeb, zatímco Sentinel nezachytil stop/start události rsyslogu kvůli jeho vypnutí a následnému socket‑based restartu.
Chování odpovídá reálným útokům, kdy útočník vypne logování, aby skryl svou aktivitu.
