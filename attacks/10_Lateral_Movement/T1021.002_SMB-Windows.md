## 🛡️ MITRE ATT&CK – Lateral Movement: SMB / PsExec (T1021.002)

### 🎯 Tactic: Lateral Movement (TA0008)

### 🔧 Technique: T1021.002 – SMB/Windows Admin Shares

### 🖥️ Platform: Windows (Windows Server, Windows 11)

### 📡 Detection Sources: Windows Event Log, Sysmon, Microsoft Sentinel

---

## 📝 1. Popis techniky

Útočník, který získá platné přihlašovací údaje (username + heslo), může provést laterální pohyb mezi Windows systémy pomocí SMB a admin shares (např. ADMIN$, C$).

Jednou z nejčastějších technik je použití nástroje **PsExec**, který:

- autentizuje se přes SMB (port 445)
- nahraje binárku na cílový systém
- vytvoří službu (PSEXESVC)
- spustí vzdálený proces jako SYSTEM

Tato technika je velmi častá v reálných útocích i v penetračním testování.

## 🧨 2. Útok (simulace)

### 🔥 2.1 Laterální pohyb pomocí PsExec (Impacket)

- vytvoří se služba **PSEXESVC**
- útočník získá vzdálený shell jako **NT AUTHORITY\SYSTEM**

### ☁️ Microsoft Sentinel

#### 🔎 KQL – základní přehled

```kql
SecurityEvent
| where EventID in (4624, 4672, 7045)
| project TimeGenerated, Computer, EventID, Account, Activity
| sort by TimeGenerated desc
```

#### 🔎 KQL – detekce PsExec služby

```kql
SecurityEvent
| where EventID == 7045
| where ServiceName contains "PSEXESVC"
| project TimeGenerated, Computer, ServiceName, Account
```

## 🧠 4. Analyst Notes (SOC pohled)

Laterální pohyb přes PsExec má velmi typický pattern:

Logon Type 3 (síťové přihlášení)
následně admin práva (4672)
vytvoření služby (7045)

Tato kombinace je silný indikátor laterálního pohybu.

Další indikátory:

SMB komunikace na portu 445
vytvoření služby PSEXESVC
spuštění procesu jako SYSTEM

Doporučená reakce:

ověřit zdroj přihlášení (IP, účet)
zkontrolovat další pohyb v síti
prověřit, zda nebyly použity sdílené credentials
izolovat kompromitovaný stroj


## 🛡️ 5. Hardening / Remediace

omezit přístup k admin shares (ADMIN$, C$)
segmentovat síť (omezit SMB komunikaci)
používat LAPS pro lokální admin účty
monitorovat Event ID 7045 (vytváření služeb)
omezit použití nástrojů jako PsExec
implementovat MFA


## 🏁 Závěr

Simulace demonstruje techniku T1021.002 – SMB/Windows Admin Shares jako základní laterální pohyb.

Útočník využil platné přihlašovací údaje k přístupu na další Windows systém a spustil vzdálený proces pomocí PsExec.

Logy ve Windows Event Logu a Sysmonu jasně ukazují typický pattern laterálního pohybu a jsou vhodné pro tvorbu detekcí v SIEMu.
